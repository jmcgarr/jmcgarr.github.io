Guide to Setting up SpecFlow and Selenuim

March 17, 2012

Photo credit: https://www.flickr.com/photos/spacex/44451125454/

The goal of this post is capture the process I went through for putting together a Behavior Driven Development sample project for a .NET team that I am coaching.  The tool I will be using is Specflow, a .NET BDD tool that works with Cucumber's Gherkin format.  Since the project would be an example of writing executable specifications against a web application, I will also be integrating Selenium/WebDriver 2.0 into the example.  I hope this tutorial helps you get started.

Environment Setup

The first part is to make sure your developmennt environment has all of the right tools downloaded and installed.  Here are the list of tools you need to have installed on your development machine:

  1. Visual Studio (I am using 2010 Premium) - I am going to assume you have this working
  2. NuGet
    1. Installation - You can follow the instructions listed on their website: http://nuget.codeplex.com/
    2. Verification - Make sure you can right click on a project in VS and see 'Manage NuGet Packages...'
  3. NUnit(In order to make Specflow and Selenium work, you need NUnit.  MSTest is not supported.)
    1. Version - 2.5.10
    2. Installation -Download and run the NUnit .msi file from NUnit's website: http://www.nunit.org/index.php?p=download
    3. Verification -
  4. Specflow
  5. Restart Visual Studio
Project Setup
Setting up the project
  1. Create a new project in Visual Studio.  For now, I would make the project a C# Class Library.  For this example, I will will call the project BDD.NET.
  2. Delete the Class
  3. Right click on the project and click 'Manage NuGet Packages'.  Search for and install the following projects:
    1. NUnit
    2. Specflow
    3. Selenium WebDriver
    4. Selenium WebDriver Support Classes
  4. Create a Features folder
  5. Right click on the Features, Add > New Item > SpecFlow Feature File.  Change the name of the file to Login.feature.  Click Add.
